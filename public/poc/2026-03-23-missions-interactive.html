¿Qué es un prompt y por qué importa?

Un prompt es la instrucción que le das a un modelo de IA. La calidad de tu prompt determina directamente la calidad de la respuesta. Piénsalo como dar instrucciones a un compañero de trabajo muy capaz pero muy literal.

Anatomía de un buen prompt

Todo buen prompt tiene 4 partes:

1
Contexto — Quién eres, qué proyecto es, qué stack usas
2
Tarea — Qué necesitas exactamente, con el mayor detalle posible
3
Formato — Cómo quieres la respuesta (código, bullets, tabla, etc.)
4
Restricciones — Qué NO hacer, límites, tecnologías específicas

Ejemplo real

// ❌ Prompt malo: "Hazme un componente de login" // ✅ Prompt bueno: "Eres un dev senior de Next.js 16 con TypeScript. Crea un componente LoginForm.tsx que: - Use Supabase Auth (email + password) - Tenga validación con zod - Muestre loading state y errores - Estilo dark mode con Tailwind - NO uses 'use server', es client component"
💡 Tip de Hoku: En la intranet, los agentes ya tienen system prompts configurados. Cuando usas modo "SAP", el prompt incluye automáticamente contexto de consultoría SAP Chile. No necesitas repetirlo.

Comparando modelos

Cada modelo responde diferente al mismo prompt. Aquí un ejemplo con "Genera un componente de pricing":

⚡ Groq (0.3s)
Responde ultra rápido pero a veces superficial. Ideal para iteraciones rápidas.
🤖 Claude (1.2s)
Código más robusto, mejor estructura, maneja edge cases. Ideal para producción.

Abre el Chat de Hoku (botón 🐾 abajo a la derecha) y prueba estos 2 prompts con el mismo pedido. Compara las respuestas:
